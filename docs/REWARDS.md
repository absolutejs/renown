# Renown rewards: procedural collectibles, pets & an optional on-chain proof layer

The design for "high-level, amazing, unique, rare" rewards — backed by deep research
(ASCII art, virtual pets, procedural generation, on-chain provenance). The guiding
principle from the research: **the seed is the asset.** Every reward is a pure,
deterministic function of a seed, so it reproduces identically anywhere — which is
exactly what makes it both *free to store* and *verifiable forever* (on-chain or not).

## 0. Status

- ✅ **Procedural engine** (`core/procgen.ts`) — `generate(seed)` → traits → OpenRarity
  rarity → tier (Common→Mythic) → symmetric ASCII creature (CA + mirror + parts +
  golden-ratio truecolor) → name. Deterministic, pure, tested. `renown summon`.
- ✅ Skills, celebrations (tiered, shiny/rainbow), ASCII spectacle (`core/ascii.ts`),
  hand-authored collectibles + calendar events (`core/collectibles.ts`).

## 1. Two kinds of collectibles (the content story)

**Curated drop-sets** — hand-authored, data-only, shippable in *minutes*. This is the
"pop-culture / fad / shared-moment" pipeline: when a meme breaks or an event happens,
add one data entry (`{ id, name, theme, items[], window }`) and it drops that day.
`core/collectibles.ts` already gates exclusives by calendar event; generalize `EVENTS`
to arbitrary **active windows** (date ranges + recurring) so a "Seasons / Drops" file
is the only thing you touch. Curated sets give *intent and theme*; they're finite and
collectible-complete.

**Wild (procedural) finds** — infinite, generated by `core/procgen.ts`, seeded by the
**real earning event** (e.g. `keccak/xmur3(githubId : repo : commitSHA : nonce)`). Each
is unique, its rarity falls out of the engine, and **1-of-1 / mythic-aura** finds are
the "incredibly rare to find" payoff. Because the seed is the real commit, the find has
built-in provenance. Next step: a `rollWild()` alongside `rollDrop()` in the commit path
that occasionally yields a procedural creature; store its **seed** (not the render) in
state; show both curated + wild in `renown collection`.

## 2. Pets (on the same engine)

A pet is a procedural creature whose **genome = the player's `playerId`** (stable →
one unique pet per dev, identical across machines), projected over live `State`:

- **Growth/evolution** gated by `totalLevel(skillXp)` (Pokémon XP gates); **branching
  lineage** by *which* skill bucket dominates (Sonic Chao "influence slider" math over
  renown's skill buckets — a Testing-heavy dev grows an Owl, an OSS-heavy dev a Phoenix).
- **No death** (the busy-dev trap): graceful decay → Drowsy → Asleep 💤 → Hibernating;
  streaks/bond never reset, only mood. Wake-up reuses the celebration parade + greet.
- **Care = zero new actions**: commits feed it, rare drops become accessories
  (ERC-6551-style "the pet owns its loot"), a 99 evolves it (fires the ASCII spectacle).
- Lives as a 1-line glyph+mood in the HUD and a full sprite in a TUI "Companion" tab.
- Build: `core/pet.ts` (pure projection + `derivePet`/`sprite`), `State.pet`, `ensurePet`.

## 3. Animation upgrade (phase 4+)

Current ASCII is 5-row block font + scene frames. To go further (research-backed):
- **Half-block** (`▀` fg+bg = 2 truecolor pixels/cell) for color sprites; **braille**
  (`drawille-canvas`, 2×4 dots/cell) for vector FX/particles/rings.
- A **diff'd double-buffer** (shadow `Cell[][]`, emit only changed cells) on the alt
  screen, ~12–15fps for banners / ~30fps full-screen — never `clear+reprint` (flicker).
- Optional: `chafa`(-wasm) for real image→art; kitty/iterm/sixel when detected.

## 4. The optional on-chain proof layer (tasteful, earned, never required)

**Blockchain is an opt-in *proof layer*, never a dependency.** The whole game runs
off-chain. The chain is an opt-in "mint" that turns an earned item into a permanent,
verifiable, **soulbound** 1/1. Architecture (all research-sourced):

- **The seed is the on-chain record.** Store only `seed + generatorVersion (+ tier)`;
  re-derive the art anywhere (Autoglyphs / Art Blocks / Chain Runners model). Tiny gas,
  re-derivable forever. Optionally an on-chain `tokenURI()` SVG for marketplace previews
  (Nouns RLE→rects style).
- **Soulbound = earned, not bought.** Non-transferable (ERC-5484 address-bound, or
  ERC-5114 bound to the pet). No secondary market, no floor, no speculation — the
  anti-financialization thesis of Vitalik's SBT paper. (Prior art: GitPOAP does
  blockchain-for-dev-contributions reasonably; we improve it by being soulbound + fully
  on-chain generative + giving pets ERC-6551 inventories.)
- **Provable rarity, two-tier.** Variety traits derive deterministically from the earned
  seed (entropy = the real commit). Genuine status 1/1s use **Chainlink VRF** (provably
  fair, not re-rollable) + on-chain `mapping(seed=>used)` uniqueness + supply cap. Avoid
  naive `blockhash` for status rolls (mint-and-revert / validator withholding).
- **Gasless & walletless.** Chain: **Base** (sub-cent, first-party paymaster). GitHub
  OAuth → embedded smart wallet (Coinbase Smart Wallet / Privy) → mint via ERC-4337 +
  paymaster + EIP-712 lazy/sponsored minting. The dev never buys a token, sees gas, or
  handles a seed phrase. Self-custody export optional.
- **How it dodges every NFT criticism**: PoS+L2 ≈ a few web searches of energy;
  soulbound kills speculation; no sale = nothing to rug; "right-click-save" misses the
  point (the value is provenance + the asset *is* the on-chain seed); embedded wallets
  fix custody. Honest tradeoff: a sponsored paymaster is an ongoing (sub-cent/mint) cost
  and a partial centralization point you can decentralize later. **For most players an
  off-chain signed credential gives 95% of the value with zero crypto — make the chain a
  tasteful opt-in for those who want permanent third-party-verifiable provenance.**

## Roadmap

1. ✅ Procedural engine (keystone).
2. **Wild procedural drops** + generalized curated drop-sets/seasons (the pop-culture pipeline).
3. **Pet** (`core/pet.ts`) on the engine — evolution, lineages, no-death, Companion tab.
4. **Animation upgrade** (half-block/braille diff-buffer) for evolutions & 1/1 reveals.
5. **On-chain proof layer** (opt-in): Base + soulbound + seed-is-asset + VRF for true 1/1s.
